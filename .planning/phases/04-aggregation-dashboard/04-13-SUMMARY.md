## Accomplishments

- Added `fetchR2<T>()` with a hand-rolled module-level `PromisePool(8)`.
- Added `useManifest`, `useBucket`, `useBuckets`, and `useStatus` TanStack Query hooks.
- Added `pickTier(windowDays)` boundaries: `<= 1d -> q15`, `<= 7d -> h1`, beyond -> `d1`.
- Added `LOG_BIN_EDGES` mirrored from `crates/event-schema/src/log_bins.rs` and `binLabel(idx)`.
- Added a TypeScript RFC 8785-style canonicalizer and a Rust-generated CLI payload byte fixture.

## R2 Types

`apps/frontend/src/lib/r2.ts` exports:

- `Manifest`: `{ schema_version: "v1", last_updated_ts, tiers: { q15, h1, d1 } }`
- `BucketEnvelope`: `{ schema_version: "v1", bucket_ts, tier_resolution, bin_edges, cells }`
- `StatusJson`: `_status.json` chrome data with cron timestamps, `ingest_health`, lifetime event count, and fuzzy 30d contributor count.

## Canonical Fixture Regen

Regenerate the frontend canonical byte fixture with:

```bash
cargo run -p xtask -- gen-canonical-fixture \
  apps/frontend/src/__tests__/canonical-fixtures/cli-dryrun.json \
  apps/frontend/src/__tests__/canonical-fixtures/cli-dryrun-expected-bytes.txt
```

The Vitest gate parses `cli-dryrun.json`, canonicalizes its `payload` with `canonicalize`, hex-encodes the bytes, and compares against `cli-dryrun-expected-bytes.txt`.
